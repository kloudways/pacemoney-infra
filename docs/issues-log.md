# Issues log

All issues encountered during Phases 1-5 of the Pace Money build.

| Issue | Phase | Symptom | Root cause | Fix | Prevention |
|-------|-------|---------|-----------|-----|-----------|
| `python3-venv` missing on Jenkins | 3 | `python3 -m venv .venv` failed during Unit Tests stage | Ansible common role did not install `python3-venv` or `python3-pip` | Added both packages to `ansible/roles/common/tasks/main.yml` and installed live via SSH | Include the full Python toolchain in the base Ansible role |
| Jenkins `ec2:DescribeAvailabilityZones` denied | 5 | kops failed to export kubeconfig during Helm Deploy | Jenkins IAM inline policy did not include `ec2:DescribeAvailabilityZones` | Added the permission and applied via `terraform apply -target=aws_iam_role_policy.jenkins` | Provision the full kops read-only permission set upfront |
| Jenkins ELB and Auto Scaling permissions denied | 5 | kops could not read cluster state; multiple AccessDenied errors for `elasticloadbalancing:Describe*` and `autoscaling:Describe*` | Jenkins IAM policy missing ELB v2 and Auto Scaling read permissions required by kops | Added all missing Describe* permissions in one batch and applied via Terraform | Include the complete set of kops-required read permissions in the initial IAM policy |
| RDS not reachable from kops nodes | 5 | Pods crashed silently (exit code 137, empty logs); liveness probe reported `connection refused` | The RDS security group allowed port 5432 only from `app_node_sg`. kops nodes are not assigned `app_node_sg` (kops manages its own security groups), so the TCP connection to RDS hung before uvicorn could start | Added a second ingress rule to `pacemoney-rds-sg` allowing port 5432 from the VPC CIDR (10.1.0.0/16) | Ensure security group rules cover all compute that needs database access, including kops-managed node security groups |
| kops state bucket blocks `terraform destroy` | 5 | `terraform destroy` failed with `BucketNotEmpty: The bucket you tried to delete is not empty. You must delete all versions in the bucket` | The kops state bucket has versioning enabled; `terraform destroy` cannot delete a versioned bucket with objects | Removed the bucket from Terraform state with `terraform state rm aws_s3_bucket.kops_state` and left it in AWS (it costs less than $0.01/month and is reused on the next startup) | Document this as an expected teardown step |
| ECR repository blocks `terraform destroy` | 5 | `terraform destroy` failed with `RepositoryNotEmptyException` | The ECR repository still contained images pushed during the pipeline run | Force-deleted the repository: `aws ecr delete-repository --repository-name pacemoney --force --region eu-west-2` | Add `force_delete = true` to the `aws_ecr_repository` resource, or document the CLI command in the teardown runbook |
